## How to write a good devlog entry

See [`TEMPLATE.md`](TEMPLATE.md) for the section structure.

A useful devlog entry:
- Says what you were actually trying to do (not just what the phase spec says).
- Describes the moment something clicked, or the moment you were stuck.
- Links to a photo, a serial trace screenshot, or a short video clip.
- Ends by saying what Phase N+1 will attempt.

It does not need to be long. A paragraph and a photo beats three pages with no image.
